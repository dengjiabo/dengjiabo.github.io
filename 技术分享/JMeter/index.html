3、JMeter基础

3.1、测试计划

测试计划是所有测试的根节点，一次测试只能有一个测试计划。

是使用 JMeter 进行测试的起点，它是其它 JMeter 测试元件的容器。

3.2、线程组

代表一定数量的并发用户，它可以用来模拟并发用户发送请求。实际的请求内容在Sampler中定义，被线程组包含。一个测试计划中至少要有一个线程组。

  • 线程数（threads）

指虚拟用户数，默认的输入是“1”，则表明模拟一个虚拟用户访问被测系统，如果想模拟100个用户，则此处输入100。

  • Ramp-Up Period (in seconds)

虚拟用户增长时长。比如要模拟100个虚拟用户，设置该时间为60秒，则表示在60秒内达到预设的100个虚拟用户。

  • 循环次数

该处设置一个虚拟用户做多少次的测试。默认为1，意味着一个虚拟用户做完一遍事情之后，该虚拟用户停止运行。如果选中“永远”，则意味着测试运行起来之后就根本停不下来了，除非你把它强制停止。

3.3、监听器

负责收集测试结果，同时也被告知了结果显示的方式。常用监听器有如下几种：

  • 查看结果树

可以较为详细的看请求参数，响应数据等数据。

  • 聚合报告

3.4、逻辑控制器

可以自定义JMeter发送请求的行为逻辑，它与Sampler结合使用可以模拟复杂的请求序列。

  • 简单控制器

分组功能，不具备逻辑控制。

  • 循环控制器

指定其子节点运行的次数，可以使用具体的数值，如设置为五次；也可以使用变量。Forever选项：勾选上这一项表示一直循环下去。如果同时设置了线程组的循环次数和循环控制器的循环次数，那循环控制器的子节点运行的次数为两个数值相乘的结果。

  • 仅一次控制器

在测试计划执行期间，该控制器下的子结点对每个线程只执行一次，登录场景经常会使用到这个控制器。

注意：将仅一次控制器作为循环控制器的子节点，只执行一次；若线程组循环次数为5，循环控制器循环次数为3，仅一次控制器作为循环控制器的子节点,此时执行5次。

  • 事务控制器

会产生一个额外的采样器，用来统计该控制器子结点的所有时间。

3.5、断言

可以用来判断请求响应的结果是否如用户所期望的。它可以用来隔离问题域，即在确保功能正确的前提下执行压力测试。这个限制对于有效的测试是非常有用的。

3.6、配置元件

维护Sampler需要的配置信息，并根据实际的需要会修改请求的内容。

  • HTTP Request Defaults（HTTP请求默认值）

该属性管理器用于设置其作用范围内的所有HTTP默认值，包括：HTTP请求的host、端口、协议等。

  • HTTP Cache Manager（HTTP缓存管理器）

用于模拟浏览器的Cache行为，在Test Plan增加该属性管理器后，Test Plan运行过程中会使用Last-Modified、ETag和Expired等决定是否从Cache中获取相应的元素。

  • HTTP Cookie Manager（HTTP Cookie管理器）

用于管理Test Plan运行时的所有Cookie（存储在用户本地终端上的数据）。HTTP Cookie Manager可自动储存服务器发送给客服端的所有Cookie，并在发送请求时附加上合适的Cookie；同时，也可在HTTP Cookie Manager中手工添加一些Cookie，这些手工添加的Cookie会在发送请求时被自动附加到请求中。

  • HTTP Header Manager（HTTP头信息管理器）

用于定制Sampler发出的HTTP请求的请求头文件的内容。不同的浏览器发出的HTTP请求具有不同的Agent，访问某些有防盗链的页面时，需要正确的Refer等，此时则需要通过HTTP Header Manager来保证发送的HTTP请求时正确的。

3.7、前置处理器和后置处理器

负责在生成请求之前和之后完成工作。前置处理器常常用来修改请求的设置，后置处理器则常常用来处理响应的数据。

  • 正则表达式提取器

可以通过正则表达式进行匹配，在响应数据中提取某个字段。一般在进行参数化关联的时候会用到此后置处理器。如：通过正则表达式提取器处理登录token认证问题。

3.8、定时器

负责定义请求之间的延迟间隔。

  • 简单定时器
  • 高斯随机定时器

两种定时器都可以用来模拟用户操作过程中的thinktime。

3.9 取样器

通过 sampler 取样器模拟一次实际的请求。

4、脚本录制

4.1、通过JMeter代理服务器录制脚本

（1）、开启代理服务器的路径

打开JMeter > 文件 > template > create > 打开 WorkBench 下面的代理服务器，点击代理服务器下的启动按钮。

在启动过程会提示生成CA证书，选择确定即可。

（2）、设置浏览器代理

开启代理服务器后，还需要在浏览器中配置代理。不同浏览器配置代理的过程有所不同，此处以火狐浏览器为例进行讲解。

打开火狐浏览器 > 选项 > 在常规中找到网络代理，点击设置 > 在弹出页面选择手动代理设置，代理的端口配置成JMeter启动的代理服务器对应的端口，勾选为所有协议使用相同的代理服务器 > 确定设置。

完成上面的配置之后，在火狐浏览器中输入要录制脚本的网站，按照预设场景进行录制即可，录制过程中可以看到JMeter中会自动生成对应的录制脚本。

优点：简单方便，JMeter自带。

缺点：如果要测试的网站是HTTPS，那么录制过程可能会报错，支持HTTP。

4.2、通过BlazeMeter进行录制

该插件是google浏览器中免费的插件，首先需要在google浏览器应用市场中搜索安装该应用，然后需要在BlazeMeter官网注册账号进行登录即可正常使用。此处也可以使用google账号进行登录。

使用步骤如下：

右上角点击选择安装的blazemeter插件 > 在弹出的操作界面中输入测试计划名称（后续导出的jmx脚本会以此来命名）> 点击中间的红色启动按钮启动录制 > 在浏览器中打开要录制的网站进行录制 > 录制完毕后点击红色停止框停止录制 > 在blazemeter控制界面选择 eidt JMeter script > 在弹出的Editer页面点击上方的 .jmx 格式导出jmx格式脚本

优点：无需处理HTTP、HTTPS问题，会自动进行简单的参数化

缺点：需要翻墙安装插件

5、JMeter参数化

参数化方便进行参数配置的修改，只需要在配置文件中进行配置，而不需要每次要修改配置时打开JMeter、打开JMeter脚本文件再进行修改。下面罗列了一些常用的参数化方法：

5.1、正则表达式提取器

通过正则表达式提取器对前一个接口中的返回参数进行提前，然后在后面的参数中进行调用。这种方法被称为关联。例如：在登录的时候，需要获取返回的token值，之后的接口查询过程中会使用这个token进行认证，如果不传递这个token，接口会返回401报错。

5.2、CSV Data Set Config

CSV Data Set Config 可以从指定的文件，一般是文本文件中一行一行的提取文本内容，根据分隔符拆解这一行内容，并把内容和变量名对应上，然后这些变量名就可以供取样器引用了。用户登录的账号密码可以通过这种方式进行传递、创建用户时使用的账号密码等信息也可以通过此种方式进行传递。

5.3、用户定义的变量

线程组右键添加–>配置元件–>用户定义的变量。

可以在出现的用户定义变量界面添加用户自定义变量，以及为自定义变量设置变量值。但是此种方法相较于上面两种的缺点在于如果要对变量的值进行修改只能通过jmeter UI界面进行修改，不能通过修改配置文件的方式来完成。

5.4、通过函数助手生成所需参数

择菜单栏 选项–>函数助手对话框，弹出函数助手框。此处重点介绍 __RandomString 函数。

应用场景举例：进行用户注册，要求输入的用户手机号不能重复，那么可以考虑前3位号段固定，那就对手机号后8位进行随机。
